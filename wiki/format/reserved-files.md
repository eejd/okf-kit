# Overview

Two filenames are reserved and MUST NOT be used for concept documents: `index.md` and `log.md`. They are structural, not knowledge: `index.md` is a generated directory listing that powers tree navigation and progressive disclosure; `log.md` is an optional chronological history. Because they are not concepts, the validator skips the normal frontmatter/type checks for them.

# Definition

- **`index.md`** — body-only, grouping its directory's child concepts under `# <Type>` sections (title + description each), plus a `# Subdirectories` section. Only the **bundle-root** `index.md` may carry frontmatter, and only `okf_version`. These are regenerated by [index module](/core/index-regen.md); missing `index.md` files are not an error — a consumer may synthesize them on the fly.
- **`log.md`** — optional, with ISO-8601 date headings and a leading bold word convention (`**Update**`, `**Creation**`, `**Deprecation**`). Carrying frontmatter is reported as info only.

Forward-compat note: a **nested** `index.md` that carries frontmatter is reported as **info, not error** — it is the likely marker for future multi-level (sub-bundle) support. Every directory containing a concept in its subtree gets an `index.md` on regeneration.

# Examples

A regenerated root index looks like:

    # Table
    * [users](users.md) - User accounts.
    # Subdirectories
    * [metrics](metrics/index.md)

See [Conformance](/format/conformance.md) for how reserved files interact with the severity policy.
